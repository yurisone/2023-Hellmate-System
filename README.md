# 👷‍♂️ 항만 용접 작업자 안전을 위한 스마트 용접 헬멧 시스템 (2023.04 ~ 2023.11)

항만 및 선박 건조 작업장의 밀폐 공간에서 발생하는 화재 및 질식 사고를 예방하기 위해 작업자의 건강 상태와 주변 환경을 실시간으로 모니터링하는 **IoT 기반 다중 센서 스마트 헬멧 시스템입니다.**

아두이노(Arduino)와 다수의 환경/생체 센서를 연동하여 작업장의 위험 요소를 실시간으로 수집하며, 블루투스(Bluetooth) 통신을 통해 모바일 애플리케이션으로 데이터를 전송합니다. 설정된 임계값(Threshold)에 따라 작업자의 상태를 3단계(안전/경계/위험)로 분류하고, 위험 감지 시 즉각적인 시청각 알림 및 119 자동 호출 등 안전 대응 로직이 동작하도록 구현했습니다.

---

## 🔧 Tech Stack

### Hardware & MCU
- **MCU**: Arduino Uno, Raspberry Pi 4B
- **Environment Sensors**: 불꽃 감지 센서, 유해 가스 센서(MQ-135/136 - CO, CO2, H2S), 온습도 센서(DHT22), 자외선 센서
- **Biometric Sensors**: 비접촉 온도 센서, 산소포화도 및 심박수 센서
- **Modules**: Bluetooth 모듈, 카메라 모듈

### Software & Protocols
- **Languages**: C (Arduino), Kotlin (Android App)
- **Protocols**: UART (Bluetooth Serial 통신)
- **Database / Backend**: Firebase

## 🚀 주요 기능

### 1. 다중 센서 기반 작업 환경 및 생체 데이터 수집
- 불꽃, 유해가스, 온습도, 자외선 등 외부 환경 요소와 작업자의 체온, 산소포화도, 심박수 등 내부 생체 데이터를 아두이노를 통해 통합 수집.

### 2. Bluetooth 기반 실시간 데이터 파이프라인
- 아두이노에서 수집된 다중 센서 데이터를 특정 구분자(Delimiter) 기반의 문자열 페이로드(Payload)로 패키징하여 블루투스 통신으로 앱에 실시간 전송.

### 3. 작업자 상태 분류 및 단계별 안전 제어 알고리즘
- 센서 측정값이 사전 정의된 임계값을 초과할 경우, 작업자 상태를 '정상-경계-위험'으로 분류하는 상태 판단 알고리즘 구현.
- 위험 상태(가스 농도 초과, 화재 감지 등) 도달 시, 쿨링 조끼 가동, 119 긴급 센터 연결, 작업 중지 음성 안내 등 단계별 인터록(Interlock) 대응 로직 수행.

## 👨‍💻 My Role

- **Bluetooth(UART) 기반 데이터 송수신 파이프라인 구축**: 아두이노의 다중 센서 배열 데이터를 하나의 문자열로 패키징하여 송신하고, 앱(Kotlin)에서 이를 수신(`onDataReceived`)하여 센서별로 분리 및 형변환하는 직렬 통신 파싱 로직 구현.
- **다중 임계값 기반 작업자 상태 분류(State Machine) 알고리즘 설계**: 온도, 가스, 심박수 등 다양한 센서의 복합적인 입력값을 기반으로 현재 작업자의 위험도를 3단계로 판별하고 예외 처리를 수행하는 조건 분기 로직 설계.
- **요구사항 정의 및 우선순위 기획**: 현장 작업자 인터뷰를 기획하고 수행하여 시스템의 핵심 방향성을 '가스/위치'에서 '밀폐 공간 화재 감지'로 재정립 및 아키텍처 설계 주도.

## 🛠 Troubleshooting

### 1. 다중 센서 데이터 직렬 통신 파싱 및 동기화 문제 해결
**문제**: 블루투스를 통해 아두이노에서 앱으로 여러 센서 데이터(온도, 습도, 가스, 심박수 등)를 동시에 전송할 때, 데이터의 경계가 섞이거나 자료형이 일치하지 않아 UI 갱신 시 오류가 발생했습니다.
<br><br>
**해결**: 블루투스 기본 통신 로직을 응용하여, 전송부(C/C++)에서 각 센서 데이터를 쉼표(`,`)를 구분자로 하는 CSV 형태의 문자열 페이로드로 패키징했습니다. 수신부(Kotlin)의 `onDataReceived` 함수에서는 수신된 데이터를 `split(",")` 함수로 분리하고, 각 인덱스에 맞춰 적절한 자료형(Double, Int 등)으로 변환하는 **데이터 파싱(Parsing) 알고리즘을 구현**했습니다. 이를 통해 통신 지연 없이 다수의 센서 값을 정확하게 분리하고 실시간으로 상태를 갱신할 수 있었습니다.

### 2. 주관적 의견 대립을 해소하는 현장 데이터 기반 의사결정 (기능 우선순위 재설계)
**문제**: 프로젝트 초기, 팀원 간에 시스템의 핵심 기능을 무엇으로 할지(가스 감지 vs 위치 추적 vs 화재 감지) 의견 충돌이 발생하여 개발 방향이 좁혀지지 않았습니다.
<br><br>
**해결**: 기술적인 논쟁보다 '실제 사용 환경에서의 Needs'가 중요하다고 판단하여, 항만 현장 작업자를 직접 섭외해 인터뷰를 기획하고 진행했습니다. 인터뷰 결과, "선박 내부 밀폐 공간에서의 불꽃 튐으로 인한 화재"가 가장 치명적이라는 현장 데이터를 얻을 수 있었습니다. 이 객관적 근거를 바탕으로 **화재 및 온도 감지를 최우선 핵심 제어 알고리즘으로 설정하도록 시스템 구조를 재설계**하여 팀의 합의를 이끌어냈고, 결과적으로 현장 적합성이 높은 펌웨어 로직을 완성할 수 있었습니다.
